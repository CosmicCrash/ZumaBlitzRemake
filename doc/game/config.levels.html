config/levels/level_*.json

Level files contain information about sphere speeds, colors, etc.

Useful vocabulary: We say the player is in danger when the spheres are close to the end point of the path and the pyramid starts blinking.

  • The root entry.
    • map: The map name. Specifically, the name of the folder in the "maps" folder of the game.
    • music: Which soundtrack should be played throughout the level.
    • dangerMusic: Which soundtrack should be played during the danger phase.
    • ambientMusic: Which soundtrack should be played throughout the entire level, including before starting and after winning, independently of the level and danger soundtracks.
    • powerupGenerator: I'm not sure if this is used anymore.
    • gemGenerator: The collectible generator used to generate loot from successfully broken scarabs.
    • colorGeneratorNormal: The Color Generator used to generate colors for the shooter throughout this level.
    • colorGeneratorDanger: The Color Generator used to generate colors for the shooter when in danger.
    • shooter: Defines the Shooter. If not present, the shooter named default will be used along with its default movement.
      • name: Which Shooter will be used for this level.
      • movement: If defined, overrides the default movement of this Shooter. See below.
    • matchEffect: The Match Effect to be used for this level, during normal matches.
    • target: How many Spheres need to be destroyed to fill the Level's objective. Deprecated.
    • pathsBehavior: Defines the behavior of paths. The n-th entry of this list corresponds to the n-th path. If this list is shorter than the number of paths, this list will wrap around during the behavior assignment process. If this list is longer than the number of paths, all subsequent entries are discarded.
      • A single path behavior.
        • colors: A list of integers. These are the colors that will be assigned to the newly created Spheres on the path.
        • colorStreak: The probability of the next color being the same as the previous one in sequence.
        • spawnRules: How the sphere chains should be spawned. Check the possible types below.
        • spawnDistance: How far the path must be clear for a new Sphere Chain to spawn.
        • dangerDistance: How far into the path the spheres have to be in order for the player to be in danger.
        • speeds: A list of speeds which will translate to the actual speed on any point of the path.
          • A single speed graph node.
            • distance: How far into the path the given speed should be applied.
            • speed: The desired speed of the sphere trains, in pixels per second.
            • transition: Defines the transition between this and the next node on the speed graph. By default, linear transition is applied.
              • type: I don't know if this works.

Possible shooter movement types (move this to Shooter later on)

Debug information: this can be checked in Shooter:update(), Shooter:getInitialPos() and Shooter:getInitialAngle().

linear

The Shooter will be moving along a horizontal line.

  • The shooter movement.
    • type: linear
    • xMin: The minimum X position this Shooter can be at.
    • xMax: The maximum X position this Shooter can be at.
    • y: The Y position this Shooter will be at.
    • angle: The shooting (and the entire shooter) angle in degrees. 0 is up.

circular

The Shooter will be locked in place, but being able to rotate around.

  • The shooter movement.
    • type: circular
    • x: The X position of the Shooter.
    • y: The Y position of the Shooter.

Possible spawn rule types

Debug information: this can be checked in SphereChain:new().

waves

The Sphere Chains will be prebuilt with the given number of spheres. The scarab is not counted.

  • The spawn rules.
    • type: waves
    • amount: How many spheres should be generated in each wave.

continuous

The Sphere Chains will not end, and new spheres will be generated dynamically. The generation will end when either the level target is fulfilled, or the level is lost.

  • The spawn rules.
    • type: continuous
