config/level_set.json

Contains information about level order and checkpoints.

A checkpoint is a place, where the player can start a new game.

  • The root entry.
    • level_order: A list which contains level entries. Levels will be played in this order. More info below.
    • checkpoints: A list of integers, containing index numbers corresponding to level IDs in the level_order field.
    • startCheckpoints: A list of checkpoint IDs, which are unlocked upon profile creation.

Possible level order entry types

Debug information: this can be checked in Profile:getLevelName(), Profile:incrementLevel(), Profile:generateLevelID() and a number of other functions in Profile class.

level

A single level to be played.

  • A single level entry.
    • type: level
    • level: A level ID to be loaded and played.
    • name: A level name.
    • unlock_checkpoints_on_beat: If specified, after beating this entry, checkpoints with IDs corresponding to numbers in this list will be unlocked.

randomizer

A group of levels to be played in random order.

  • A single level entry.
    • type: randomizer
    • pool: A list of level IDs to be chosen from.
    • names: A list of level names. It is not shuffled, and this list should have the same lenth as count.
    • count: How many levels are to be picked from the pool.
    • mode: How the levels will be randomized. There are three possible values:
      1. repeat - Every next level is simply drawn at random from the list.
      2. no_repeat - Same as above, but an entry is removed from the pool once it's been picked.
      3. order - Chooses count entries from the pool, without changing their order.
      4. Both no_repeat and order modes need the pool size to be greater or equal to the count.
    • unlock_checkpoints_on_beat: If specified, after beating this entry, checkpoints with IDs corresponding to numbers in this list will be unlocked.
