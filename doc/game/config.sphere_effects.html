config/sphere_effects/*.json

Defines all Sphere Effects in the game.

A sphere effect can be applied to any sphere. After a certain time, spheres "infected" with a given effect will be destroyed.

A sphere can also infect neighbors, if it has been configured to do so. Until the infection size gets to 0, the infection time is counted each time. When it gets to 0, the main destruction timer starts.

When the sphere effect timer gets to 0, all spheres which have that same effect and were caused by the same sphere will be destroyed.

  • The root entry.
    • particle: A persistent particle packet which will be applied to any affected sphere.
    • time: After how many seconds the spheres will be destroyed.
    • infection_size: How many times the effect expansion should occur.
    • infection_time: Time between the expansions, in seconds.
    • apply_sound: A sound event to be played when a sphere gets this effect.
    • destroy_sound: A sound event to be played when spheres afflicted with this effect are destroyed.
        • Can be set to hardcoded, in that case the destroy sound from a game module is prioritized.
    • destroy_font: A font to be used for the score value displayed on the screen.
        • Can be set to hardcoded, in that case the font used is of the "cause" sphere's matchFont value.
    • destroy_particle: A one-time particle packet used when a sphere with this effect is destroyed.
    • destroy_collectible: The collectible generator to be used when spheres with this effect are destroyed.
    • level_loss_protection: If set to true, a group with spheres with this effect applied will not trigger a level loss when past the path's end node.
    • immobile: If true, the spheres affected with this effect will stop.
    • fragile: If true, the spheres affected with this effect can be destroyed by shooting through them.
    • can_boost_combo: If true, the level combo value may rise after destroying a group with this effect.
    • can_boost_chain: As above, but regarding the chain combo value.
    • apply_chain_multiplier: If true, the score value obtained from destroying the affected spheres will be multiplied by the current chain combo value.
    • can_keep_combo: Whether shot spheres with this effect should not be counted towards combo reset.
    • cause_check: When set to true, two groups of spheres of the same sphere effect which happen to collide will behave as one, ignoring the "cause" sphere.
