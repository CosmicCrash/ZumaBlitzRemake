config/collectibles/*.json

Defines all collectibles in the game.

Each collectibles is stored in a separate file, located in config/collectibles/*.json. The name of the file is the same as ID of that collectible.

To add a new collectible, simply create a new file in the aforementioned folder.

Note: Collectibles are not rendered as sprites, but rather as particles. The particle packet used to render a collectible needs to be persistent.

  • The root entry.
    • particle: A persistent particle packet which will follow the collectible entity.
    • pickupParticle: A one-time particle packet which will be launched when the collectible is catched.
    • spawnSound: A sound event which will play when the collectible is spawned.
    • pickupSound: A sound event which will play when the collectible is collected.
    • pickupName: A label which will appear above the shooter when the powerup is collected.
    • pickupFont: A font which will be used for the label.
    • effects: A list of effects to be applied when the powerup is collected. More info below.
    • dropEffects: A list of effects to be applied when the powerup is not collected and falls out of the screen. More info below.

Possible powerup effects

Debug information: this can be checked in Level:applyEffect().

replaceSphere

Replaces the current sphere the shooter has with a defined sphere. This can be used to spawn otherwise unobtainable bonus balls, i.e. wild or lightning.

  • A powerup effect.
    • type: replaceSphere
    • color: A sphere ID of a sphere the shooter should get.

multiSphere

Spawns a certain amount of a defined sphere to the shooter. This can be used to i.e. spawn 10 daggers the player can shoot anywhere they want.

  • A powerup effect.
    • type: multiSphere
    • color: A sphere ID of a sphere the shooter should get.

speedShot

Replaces the default shooting speed and spawns a speed shot beam effect.

  • A powerup effect.
    • type: speedShot
    • time: The duration of the effect in seconds.
    • speed: A new speed of the shot balls, in pixels per second.

speedOverride

Changes the speed of spheres on the board.

  • A powerup effect.
    • type: speedOverride
    • speedBase: A value to be added to the multiplied sphere speed.
    • speedOverride: A multiplier by which the nominal path speed will be multiplied. The formula for the final speed is: speedBase + (pathSpeed * speedOverride).
    • decceleration: A sphere decceleration rate which will be applied alongside speed changes.
    • time: The duration of the effect in seconds.

destroyAllSpheres

Clears the board from all spheres.

  • A powerup effect.
    • type: destroyAllSpheres

destroyColor

Destroys the spheres of a specified color.

  • A powerup effect.
    • type: destroyColor
    • color: The color of spheres to be destroyed.

spawnScorpion

Spawns a scorpion on the board.

  • A powerup effect.
    • type: spawnScorpion

lightningStorm

Replaces the current sphere the shooter has with a defined sphere. This can be used to spawn otherwise unobtainable bonus balls, i.e. wild or lightning.

  • A powerup effect.
    • type: lightningStorm
    • count: The amount of strikes.

changeGameSpeed

Changes the game speed to a defined value for a specified period of time.

  • A powerup effect.
    • type: changeGameSpeed
    • speed: The game speed multiplier.
    • duration: The duration of the effect in seconds. It is not affected by the game speed itself.

setCombo

Sets the current level combo to a specified value.

  • A powerup effect.
    • type: setCombo
    • combo: The specified value.

grantScore

Grants a certain amount of score to the current player.

  • A powerup effect.
    • type: grantScore
    • score: The score to be given.

grantCoin

Grants a single coin to the current player.

  • A powerup effect.
    • type: grantCoin

incrementGemStat

Adds one to the gem pickup statistics for the current level.

  • A powerup effect.
    • type: incrementGemStat
